# Задания курсовой работы по цифровой и микропроцессорной технике

## ВНИМАНИЕ:  
Этот репозиторий предназначен для выполнения и сдачи студентами курсовой работы на базе микроконтроллера PIC18F2520 с использованием языка программирования C.  
Каждый студент должен работать в своей отдельной ветке.  
В финальной версии вашей ветки должны находиться:  
- Отчёт по курсовой работе (в формате PDF или DOCX)  
- Электрическая схема (в формате PDF, PNG или JPG)  
- Схема алгоритма работы программы (в формате PDF, PNG или JPG)  
- Исходный код программы на языке C (отдельная папка source)

## СОДЕРЖАНИЕ:

1. Как получить доступ к репозиторию  
2. Как создать свою ветку  
3. Как загружать файлы в свою ветку  
4. Как обновлять свою ветку в течение семестра  
5. Что делать, если возникли конфликты или ошибки  
6. Требования к структуре и содержанию файлов  
7. Важные напоминания и правила  
8. Контакты для помощи

### 1. КАК ПОЛУЧИТЬ ДОСТУП К РЕПОЗИТОРИЮ

Шаг 1. Убедитесь, что у вас есть аккаунт на GitHub (https://github.com). Если его нет — зарегистрируйтесь.

Шаг 2. Перейдите по ссылке на репозиторий курсовой работы.

Шаг 3. Нажмите кнопку "Fork" в правом верхнем углу страницы. Это создаст копию репозитория в вашем личном аккаунте.

Шаг 4. Скопируйте ссылку на ваш форк (она будет выглядеть как: https://github.com/ваш_логин/название_репозитория.git).

Шаг 5. Установите Git на свой компьютер, если он ещё не установлен:  
   - Windows: https://git-scm.com/download/win  
   - macOS: https://git-scm.com/download/mac  
   - Linux: используйте пакетный менеджер вашей системы (например, `sudo apt install git`)

Шаг 6. Откройте терминал (или Git Bash на Windows) и выполните команду:  
   ```
   git clone https://github.com/ваш_логин/название_репозитория.git
   ```

Шаг 7. Перейдите в папку проекта:  
   ```
   cd название_репозитория
   ```

### 2. КАК СОЗДАТЬ СВОЮ ВЕТКУ

Каждый студент должен работать в своей ветке. Название ветки должно соответствовать вашему варианту и ФИО.  
Формат названия ветки: `variant_XX_Фамилия_Имя`  
Пример: `variant_07_Иванов_Алексей`

Шаг 1. Убедитесь, что вы находитесь в главной ветке (обычно main или master):  
   ```
   git checkout main
   ```

Шаг 2. Создайте новую ветку с вашим именем:  
   ```
   git checkout -b variant_XX_Фамилия_Имя
   ```

Шаг 3. Отправьте вашу ветку на GitHub:  
   ```
   git push -u origin variant_XX_Фамилия_Имя
   ```

Теперь у вас есть собственная ветка, в которую вы будете загружать все материалы.


### 3. КАК ЗАГРУЖАТЬ ФАЙЛЫ В СВОЮ ВЕТКУ

Каждый раз, когда вы хотите сохранить прогресс (например, добавили схему, написали часть кода, обновили отчёт), выполните следующие шаги:

Шаг 1. Убедитесь, что вы находитесь в своей ветке:  
   ```
   git branch
   ```  
   Текущая ветка будет отмечена звёздочкой (*). Если вы не в своей ветке — переключитесь:  
   ```
   git checkout variant_XX_Фамилия_Имя
   ```

Шаг 2. Скопируйте или переместите ваши файлы в папку репозитория на компьютере.  
Рекомендуется создать внутри папки подпапки для удобства:  
   - /report/ — для отчёта  
   - /schematics/ — для электрической схемы  
   - /flowchart/ — для схемы алгоритма  
   - /source/ — для исходного кода

Шаг 3. Добавьте новые или изменённые файлы в индекс Git:  
   ```
   git add .
   ```  
   (точка означает "все файлы в текущей папке и подпапках")

Шаг 4. Сделайте коммит с описанием изменений:  
   ```
   git commit -m "Краткое описание того, что вы сделали"
   ```  
   Пример:  
   ```
   git commit -m "Добавлена электрическая схема и первая версия кода"
   ```

Шаг 5. Отправьте изменения на GitHub:  
   ```
   git push origin variant_XX_Фамилия_Имя
   ```

После этого ваши файлы появятся в вашей ветке на GitHub.


### 4. КАК ОБНОВЛЯТЬ СВОЮ ВЕТКУ В ТЕЧЕНИЕ СЕМЕСТРА

Работа над курсовой — это процесс. Вы будете вносить изменения много раз. Каждый раз повторяйте действия из пункта 3.

Рекомендации:  
- Сохраняйте изменения часто, даже если работа ещё не закончена.  
- Пишите понятные сообщения коммитов — это поможет вам и преподавателю отслеживать прогресс.  
- Не ждите последнего дня — загружайте материалы по мере выполнения.


### 5. ЧТО ДЕЛАТЬ, ЕСЛИ ВОЗНИКЛИ КОНФЛИКТЫ ИЛИ ОШИБКИ

Если вы видите сообщение об ошибке — не паникуйте. Чаще всего это решаемо.

Самая частая ошибка — вы пытаетесь сделать push, но уже изменили с другого устройства ветку на сервере.  
Решение: сначала получите изменения с сервера:  
   ```
   git pull origin variant_XX_Фамилия_Имя
   ```

Если возник конфликт (например, вы и кто-то ещё редактировали один файл) — Git сообщит об этом.  
Вам нужно будет вручную открыть файл, найти конфликтующие участки (они будут помечены >>>>>> и <<<<<<) и оставить нужную версию.  
После исправления:  
   ```
   git add имя_файла
   git commit -m "Исправлен конфликт"
   git push origin variant_XX_Фамилия_Имя
   ```

Если вы запутались — обратитесь к преподавателю или старосте группы. Не удаляйте репозиторий и не начинайте заново без консультации.


### 6. ТРЕБОВАНИЯ К СТРУКТУРЕ И СОДЕРЖАНИЮ ФАЙЛОВ

К моменту сдачи курсовой в вашей ветке должна быть следующая структура:

```
ваша_ветка/
├── report/
│   └── report.docx (или report.pdf)
├── schematics/
│   └── circuit.pdf (или .png, .jpg)
├── flowchart/
│   └── algorithm.pdf (или .png, .jpg)
└── source/
    ├── main.c
    ├── Makefile
    └── (дополнительные файлы)
```

ВАЖНО:  
- Все файлы должны быть в читаемом формате.  
- Исходный код должен компилироваться и соответствовать вашему варианту задания.  
- Схемы должны быть чёткими, с пояснениями.  
- Отчёт должен содержать титульный лист, описание работы, список использованных компонентов, анализ кода, выводы.


### 7. ВАЖНЫЕ НАПОМИНАНИЯ И ПРАВИЛА

- НЕ РАБОТАЙТЕ В ВЕТКЕ main (master) — это может сломать общий репозиторий.  
- НЕ УДАЛЯЙТЕ ЧУЖИЕ ВЕТКИ И ФАЙЛЫ.  
- НЕ ЗАГРУЖАЙТЕ БИНАРНЫЕ ФАЙЛЫ, ИСПОЛНЯЕМЫЕ ФАЙЛЫ, АРХИВЫ — только исходники и документы.  
- НЕ ЗАГРУЖАЙТЕ ФАЙЛЫ С РУССКИМИ БУКВАМИ В ИМЕНИ (лучше использовать латиницу, например: report.pdf, circuit_v1.png).  
- Следите за объёмом файлов — GitHub не принимает файлы больше 100 МБ.  
- Регулярно делайте push — не копите изменения до последнего дня.  
- Последний срок сдачи — дата, указанная преподавателем. После неё изменения не принимаются.


### 8. КОНТАКТЫ ДЛЯ ПОМОЩИ

Если у вас возникли трудности с Git, структурой, загрузкой файлов — обращайтесь:

- Преподаватель
- Староста группы
- Также можно задать вопрос в групповом чате (если есть) или на занятии.

Не стесняйтесь спрашивать — лучше уточнить сейчас, чем потерять баллы потом.

Удачи в выполнении курсовой работы!
